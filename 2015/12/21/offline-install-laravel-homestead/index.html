离线安装&配置laravel开发环境homestead

发表于 2015-12-20 | 分类于 laravel ， php

2016年9月更新

下载homestead镜像

开发laravel的本地环境当然是用官方提供的homestead vagrant镜像

优势不用多说，但是鉴于国内特殊原因，homestead通过命令行在线安装几乎是不可能完成的任务，只能选择先下载homestead镜像，再手动安装

如果你还没有homestead镜像，如果你有一台境外主机，通过主机->迅雷->下载,如果没有主机，可以上百度网盘搜索，如果还没有，请留言，我用我的主机给你个临时下载地址。主机下载地址

https://atlas.hashicorp.com/laravel/boxes/homestead/versions/0.5.0/providers/virtualbox.box

离线安装homestead

假设大家已经拿到了homestead镜像，首先确保安装了以下软件的最新版本，

  • vagrant
  • virtualbox
  • git
    ps:vmware也可以的，一方面我用的是virtualbox，另一方面virtualbox免费

以windows为例，linux大同小异

将镜像放到合适的目录然后cd到镜像所在目录

1.我们先来看看vagrant box 有哪些选项

vagrant box add -h
我们得知最主要的用法如下
Usage: vagrant box add [options] &lt;name, url, or path&gt;
既然是离线安装我们直接当然通过path来添加box

输入如下命令
vagrant box add laravel/homestead-demo virtualbox.box
看到success添加盒子就成功了

更新！！！！

如果出现版本错误，请在virtualbox.box镜像所在目录创建metadata.json

输入以下内容

然后输入以下命令添加box
vagrant box add metadata.json

可以通过如下命令查看安装的box
vagrant box list
可以看到我装了三个box

我们可以通过vagrant系列命令对box进行一些列的操作，这里不再赘述

克隆官方仓库

box安装后就需要clone laravel官方的homestead git仓库了

我先理一理 vagrant ，virtualbox ，homestead镜像，以及官方homestead git仓库的关系

virtualbox是虚拟机，(-_-|| 废话)，他最终的任务就是运行homestead虚拟机，

homestead镜像就是laravel官方为了方便开发者，将一系列的开发环境、软件打包成一个镜像供大家使用，有自己配过linux开发环境的同学就知道要弄好一个舒服的环境有多麻烦，而且版本切换，还原，代码传输都是麻烦事，vagrant的出现就是为了解决这么一个问题，vagrant可以看作是对virtualbox或vmware的一个高级封装，本质就是调用了一些virtualbox和vmware开放出来的api，vagrant官网介绍一开始就有说Vagrant stands on the shoulders of giants，所以他能站在高处通过完美的封装方便开发者，想更多了解请移步vagrant docs，homestead git仓库呢则是laravel官方对于homestead虚拟机的一些配置文件，里面有一些方便的linux脚本，这个待会儿介绍

废话说完，我们先clone 版本库
git clone https://github.com/laravel/homestead.git Homestead

配置homestead

完成以上操作我们进到clone的目录里运行init.sh

对于windows 双击也是可行的

这会在windows家目录下生成 一个名为 .homestead的文件夹

里面就是homestead虚拟机的配置文件

Homestead.yaml

参数都是很简明易懂，有一点需要注意的是rsa 公钥私钥路径一定要对，不然无法启动

没有的话通过git bash或其他linux环境输入以下命令生成
ssh-keygen -t rsa -C "you@homestead"

启动homestead

设置yaml配置文件后，我们就可以启动homestead了，进入到homestead目录(git clone的那个)，一个简单的命令
vagrant up
第一次启动会比较慢，耐心等待，如果你的硬盘是SSD那就很快了

由于vagrant进行了端口转发(保护本地常用端口)，所以ssh端口不再是熟悉的22

被转发到了2222，我们要想通过putty或者其他ssh工具连接的话地址应该填上
127.0.0.1:2222
然后方便调试我们还需要修改windows本地hosts文件

进入hosts所在目录

添加新的一列
192.168.10.10 homestead.app
到现在配置基本完成

我们ssh到homestead
cd到windows本地文件映射对应的路径，按照我上面的yaml配置文件应该为
/home/vagrant/Code
在目录下运行
laravel new helloworld

由于不可抗拒的原因只能喝杯咖啡再来看

等依赖安装完毕，我们浏览器键入homestead.app
