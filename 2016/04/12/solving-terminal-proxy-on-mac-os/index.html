MacOS下解决终端代理问题

发表于 2016-04-11 | 分类于 go ， MacOS ， 科学上网

在这个神奇的国度，每天都得花上不少的时间越过长城走向世界😂

最近转战到MacOS下面进行开发学习，我拿到一个新系统第一次事肯定是解决科学上网，通过proxychains已经能很轻松的解决绝大部分的终端代理问题。

安装homebrew

如果你才开始使用MacOS，brew作为开发是必不可少的工具，其作用类似于apt-get，yum。

安装非常简单，按下cmd+空格 输入terminal,回车打开终端

输入以下命令，中间会有一些会提示输入密码，按着提示输入就行

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

安装proxychain

有了homebrew，安装就非常简单了
brew install proxychains-ng

配置proxychain

proxychains 配置非常简单

将最后一行本用于tor匿名代理的配置修改为
socks5 127.0.0.1 1080
这样就能通过proxychains转发终端中的请求给ss

使用proxychain

我们在终端中输入如下代码，如果我们能成功访问到google说明我们proxychains已经安装成功
proxychains4 curl www.google.com
这里一定要注意proxychains从名字都能看出来是一个链式代理，如果你有设置过http环境变量会引起proxychains链式调用，会出现问题

偷个懒，加入alia

privoxychains4 这个命令太长了，虽然有补全还是很烦，时间宝贵，我们给他取个别名
