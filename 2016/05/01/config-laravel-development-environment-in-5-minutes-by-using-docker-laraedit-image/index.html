使用docker镜像laraedit在5分钟内配置laravel开发环境

发表于 2016-05-01 | 分类于 Docker ， laravel ， MacOS ， php

为什么使用docker配置laravel开发环境

  • 容器启动速度快，真的很快，容器启动时间大约为3-5秒
  • 配置简单，易修改
  • 相比vagrant更轻量级，可以同时部署多个容器

安装docker

略

pull laraedit镜像

  • 首先配置好docker的代理，详情请参见我的这篇博文为kitematic设置代理加速docker pull
  • 开启docker-machine后在可以通过在kitematic中搜索laraedit并添加镜像

  • 你也可以通过终端键入如下命令

    docker pull laraedit/laraedit`


    当然能否在5分钟内下载完都取决于你的代理服务器了，当然大家可以选择国内daoclound，灵雀云一类云服务商的镜像服务来加速，我的梯子很好用，就不去尝试了
    

    • 



配置laraedit
我们当然不是从头去配置一台服务器，我们要做的仅仅是键入命令映射端口和文件



至于端口和目录大家可以根据自己的情况填写


测试
我们通过kitematic点击EXEC开启一个laraedit的shell




进入到shell中我们先切换到bash



等laravel命令跑完后，Kitematic的web preview已经可以看见laravel的标志了




点击laravel 5区域则会通过浏览器打开此地址


退出并保存容器
我们当然不想每次都来设置一番，我们需要保存当前运行的容器实例，如果大家没有重命名的话，当前laraedit运行的容器默认命名是类似amazing_colden的一个随机字符串


接下来我们将我们修改过的laraedit保存成一个新的镜像



进一步的优化设置
我们可以将docker-machine的ip写入hosts文件，这样我们就可以通过域名访问laravel项目，而不用敲ip


回到物理机上



我们在浏览器中输入 homestead.app，已经能正常访问页面了。
