玩转phpstorm教程系列(1)创建项目

发表于 2016-08-27 | 分类于 IDE ， laravel ， php ， PHPStorm ， 工具

前言

phpstorm这个系列很久之前就有想写，一直懒着没写。写这个系列主要考虑到了中国php开发人员相对来说平均英语水平较差，不喜欢没汉化的IDE，还是喜欢文本编辑器，phpstorm相关中文教程也比较少，特别是关于xdebug以及phpunit的单元测试，我看到很多中国的php开发人员不喜欢打断点而热衷于echo、print_r 以及var_dump，更别说单元测试，可能对他们来说就是闻所未闻，在我惊愕之余其实想想主要原因还是没教程，会做的不说，不会的懒学。

我使用这款ide也差不多快两年了，一款优秀的IDE值得被大家推广、使用。希望把自己学习的一些东西分享一下，尽绵薄之力推动php在中国的标准化。

老套的开场白，工欲善其事是必先利其器。

第一节主要讲讲如何创建新项目。

准备

  • 一台装有phpstorm的电脑
    你肯定要说创建项目有什么好说的，我可以保证我讲的方式你至少有一种没有用过。

新建项目类型与区别

首先phpstorm新建项目可以分为两种

  • 没有源码，从头开建
  • 已经有源码
    至于前者比较简单，我们来看一看

通过工具栏依次 file->New Project我们就能看到下面的对话框

左边是项目类型。默认是没有类型。如果你也想创建一个Drupal工程或者wordpress插件也是可以选择的。当然用phpstorm写前端也是可以的node.js、bootstrap，angular都不在话下。

如果是默认工程，我们需要项目位置以及解释器和语言版本。

一般来说语言版本和解释器应该是一个版本才对。如果你电脑上只安装了php7.0 但是项目是5.6的这也不影响。但是如果你项目是用了php7的太空舱符号，而解释器却选择5.6运行肯定会出现问题了。

一般来新建空工程的情况比较少，新人练习语法还不错，但是我们大多数情况都是通过框架开发或者是在已有代码的基础上进行开发，所以我着重讲讲从源码新建项目。

如果你没使用过此功能，你可以先想想，从源码新建项目会遇到几种场景呢？

－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－

我们看看细心到极致的phpstorm给出了几种场景

  1. 服务器在本地，代码在服务器的web根目录下
  2. 服务器在本地，代码在本地其他地方
  3. 远程服务器，但是代码通过网络共享或者驱动挂载
  4. 远程服务器，文件可以通过FTP/SFTP/FTPS
  5. 代码在本地，服务器还没配置
    1.初学者比较常见，照着教程配置了服务器代码挂到根目录。

2.和1基本类似，也有可能是在*nix机器下面做开发。

3.这种情况最为常见，本地开发时为了保证环境一样我们采用的vagrant/docker/原生虚拟机 都采用共享目录的方式进行代码传输。

4.一般来说可以想象成在线上改代码即代码还没正式上线只是测试环境的远程服务器，强烈不建议在生产环境修改代码。

5.没什么说的

新建服务器

使用已存在的服务器还是新建服务器，既然都在看我的教程，你肯定是得新建一个才对。

注意左下角的Don’t check HTTP connection to server 如果你服务器还没搭好或者处以关机状态，你得钩上它不然连不上会报错。

首先得给你的服务器写个名字，不然到时你的服务器多了就分不清了，看你怎么取了，第二个比较重要千万别给弄错了，即你的本地硬盘和远程服务器(一般来说就是虚拟机)的共享目录，估计看我博客的人laravel用得比较多，我就拿laravel的homestead举例，你可以看到homestead的yaml配置文件folders字段你填写的哪里。

我贴上来看看

folders:
    - map: ~/dev
      to: /home/vagrant/Code

可以看到，我把我家目录下的dev文件夹全部映射到了Homestead的/home/vagrant/Code下。

显而易见对我来说我肯定填写~/dev咯

webserver root这个其实关系倒不是很大，现在测试一般都是自定义run case，不管都可以。如果你有自定义的域名就填写一个，比如laravel.app

不检查服务器http连接也最好勾上

好服务器创建到此结束。

选择项目目录

在下一步我们就可以看到，对话框列出了共享目录下的文件，让我们选择。

道理很简单，一个虚拟机不能只跑一个项目，可能多个项目都在里面同时开发，我们选择当前我们的项目。点击最上方的project root。

可以看到选择以后又出现了3个选项，这三个选项不只是这种新建项目才会存在，所以我分别详细讲讲。

  • Excluded from download
    不下载。什么东西可能不下载呢，当然是和项目无关的东西，你可以想想比如项目的一些缓存目录，一些日志目录，这些就是不用下载的地方，你可以选择它们排出掉，它们可能会很大，下得也很慢。

  • Resource Root
    资源根目录，当然是放js，css，html模版的地方咯

  • Excluded from indexing
    IDE的什么智能提示，代码跳转，补全等等神技都是索引出来的结果，可以说IDE在一定程度上做了编译器、解释器的语法分析工作。文件很多的时候索引时间就会很长，如果有一些项目文件(不只是php,js、html同样也会)你根本不可能调用到你大可排除索引。

再下一步会叫你选择你web访问地址，这种是出于什么考虑呢？

想象一下 www.laravel.app/doc 和 www.laravel.app/api 通过nginx、apache的重写规则完全可以由两个web项目组成 。可能服务器在laravel.app但是你的项目在子域名下才能访问

好一番功夫总算创建好了。

下面讲讲使用第四种使用SFTP方式创建项目。

一般来说FTP不安全很少使用，而安全版的FTPS使用也想对较少，最常见的还是基于ssh访问的SFTP(Secure File Transfer Protocol)来进行文件传输。

所以要想使用这种方法请确保你的”远程”服务器开启了sshd服务并允许外网访问。

前面步骤大同小异。只是创建服务器稍有不同。

我这里使用基于vagrant的Homestead给大家演示一下。填写上服务器地址(ip or 域名)，账号密码(一定要确保这个账号有权限访问你的代码所在的目录)。测试一下你的连接，第一次连接的话有ssh连接经验的同学就会知道这里会有一个握手密钥你要接受，不接受你当然就没得玩，就像打电话一样，A:你是谁？B:滚你mb的@＃@，你觉得A还有必要和你说下去么。

测试成功，服务器创建完成,下面的步骤就和3没啥区别了。

当然还可以通过VCS版本控制从多个仓库导出项目，但这并不妨碍上面的项目创建，因为我们的代码可能本来就已经有了版本控制，就算还没加入版本控制在项目创建完成再来配置也并无大碍。
