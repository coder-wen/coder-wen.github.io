玩转phpstorm教程系列(2)自定义设置

发表于 2016-09-03 | 分类于 IDE ， php ， PHPStorm ， 工具

这是phpstorm教程系列第2节

今天我们来讲一讲phpstorm中的自定义设置。

我讲的自定义设置主要集中在Appearance & Behavior 以及Editor两大板块，其他的设置我会在其它章节讲到。

Appearance & Behavior

  • Appearance 这下面最主要的一个选项就是IDE主题，就是两个一个default一个darcula，我一般都使用darcula比较护眼。其他选项都没什么好说的。
  • Menus and Toolbars 顾名思义主要是设置菜单及工具栏的显示及顺序，你可以自定义你喜欢的菜单和工具栏，当然一般来说默认的就是最好的了。
  • System Settings 虽然有4个子项，但对国内开发者来说最重要的还是HTTP Proxy这项，因为有些插件或者其它网络访问(如果你用AS写过Android你就知道gradle的速度还是很感人的)受阻时，填上socks5或者其它你正在使用的代理会让开发变得愉快
  • File Colors 你可以根据文件类别设置文件的颜色，比如一般来说测试就是绿色的，当然这东西我并没有用过。
  • Scopes 这个东西就比较好用了，我来着重讲讲，当项目文件比较多时，很多情况PHP程序员不仅仅是在*.php打交道，还有js、css以及其它的各种库文件，当我们在各类文件中切换的时候就非常麻烦，scope起了一个把一类文件划到一堆的作用，我们通过不同的scope就能切换到不同的文件下，免除其它文件的干扰。我来举个例子：
    添加scope首先会出现两个选项，，local当然就是本项目可用，shared就是所有项目通用，大多数情况我们当然选择本项目可用，我取个名字叫views，我会把laravel下的views文件夹设为scope，这下我就不用挨着点resource/views/。。。。,非常节约时间

include和include recursively的区别一定要知道，前者只包含此文件夹本身，后者还包括其子文件夹，当然exclude以及exclude recursively就反其道而行之了。对我们来说一般选择递归包含。点击应用，OK

后面还有三个选项，但不常用我就略过了，如果我发现了他们有什么奇效我再来补充

Editor

接着说说Editor，之所以跳过Keymap呢因为很多快捷键都要讲很长，不可能一节就讲完。我会抽出几个小节来专门讲快捷键

  • General 一般来说没什么好设置的。
  • Color & Font 最重要的部分的来了，首先我们切到Font一栏。

可以看到字体也是有个主题的，这个主题和IDE的主题不一样虽然都是darcula，我们要修改darcula的字体时得先save as备份才能改，名字随便取，默认就是darcula copy了，编程所用的字体优先考虑等宽字体(monospaced font) 我使用的是Mac自带的menlo字体，windows上面可以选择source code pro等常见编程字体，然后就是字号我个人使用的15号，字体大小主要还是结合个人习惯及屏幕大小来选，下面有预览框，感觉合适就差不多 了。

Color & Font 接下来的诸多选项都是各种粒度的颜色设置大到编辑器小到某种语言的某个警告都可以自定义设置，我也就不细说，一般来说默认的就可以满足使用。

  • Code Style 这个栏目主要是针对不同的语言制定不同的代码风格，比如C语言很多人就喜欢把花括号写在同一样，而PHP则一般换行，以及tab党和空格党的争执。很多时候不习惯别人的代码风格就会很恼火，分分钟想烧死异教徒有没有，我主要挑php的code style说一说

可以看到我圈出来的地方就是括号换行设置的地方，你可以针对命名空间、类定义以及函数后面的花括号做不同的换行设定。

接着再推荐个我个人比较喜欢的两个设定，当然合不合你胃口还得看你了

一个是return语句前空一行，这样会让函数体很清晰，不容易犯错。

还有一个就是治愈强迫症的法宝，数组定义时键值对对齐，真的很爽，你值得拥有。

其它语言的code style我就不说了，依葫芦画瓢自己摸索自己喜欢的style。

  • Inspetions，就是各种各样的检查提醒，你可以根据需要关闭某些检查或者更改他们的错误类型。这个一般来说不用去动，拼写检查的错误我会给你一种最好的解决方案。
  • File and Code Template，就是各种文件模版，我一般很少用他们，取而代之的是(动态模版)live template，所以我就略过了。
  • File Encodings一般来说都是UTF8的项目，不用去管他，如果你接到了一个GBK的项目那可以到这来设置项目编码。
  • Live Template 动态模版，由于功能过于强大。我暂时跳过，下次来讲。
  • FileTypes主要是定义一些文件后缀名关联的，一般来说也不用在意，如果你有把html当做php来编辑的需求那也可以来设置一下。
  • Copyright 如果你的代码需要加入自己的版权申明，可以把模版设置好，通过快捷键一键导入到代码头，再也不用复制粘贴了。
  • Intentions跳过，以后结合快捷键再讲，我们就讲讲Spelling和Todo其它的就没什么说的了。
  • Spelling来说主要是拼写检查，很多时候过度缩写，以及中文开发人员经常会遇到的特定名字的拼音无可避免的就会报个拼写错误的小警告波浪线，看着很是不爽。很多人索性关掉了拼写检查，这其实并不是一种明智的选择，拼写检查很大程度能避免你的bug，特别是php、python这种弱类型语言。Spelling允许你加入单词或者字典来避免拼写检查的误伤，但是每次来添加是不是感觉很麻烦，也不可能你项目才开始你就有你的拼写错误单词的字典了吧，所以提前说个快捷键，对着报拼写错误的变量alt+enter，出现了Typo:save xxxx to dictionary的选项回车，警告消失了吧，真爽是不是。
  • TODO，很多程序员会调侃TODO ＝ Never To Do，但是段子归段子，todo很多时候还是很有用的，这个选项允许你自定义todo的规则，默认是todo 和 fixme你可以添加自己的style
     

 

好了，玩转phpstorm教程系列(2)的自定义设置就讲完了，phpstorm的自定义设置当然不止这一点点，我只是照着设置菜单把常用的讲了一下，还有很多奇技淫巧等着我们去发现，敬请期待。
