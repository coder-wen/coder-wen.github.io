玩转phpstorm教程系列(3)配置基于xdebug的debug

发表于 2016-09-08 | 分类于 IDE ， laravel ， php ， PHPStorm

玩转phpstorm前面的两讲已经介绍了在phpstorm上创建项目以及一些常用的自定义设置的基本流程。

今天我来讲讲最重要也算是最麻烦的配置：基于xdebug的debug配置。

原理

首先常见的debug场景主要分两类：

  1. local server
  2. remote server
    之前讲创建项目的时候我也说了最常见的开发情景就是基于vagrant、docker一类的虚拟机服务器开发，也就是所谓的remote server，以程序员的直觉，要完成这样的debug任务肯定得在开发机器和远程服务器之间建立连接，对我们来说建立通讯的两端就是服务器上(remote server)的xdebug和我们开发机上的IDE(对我们来说就是PHPStorm,更准确的说应该是PHPStorm上的php相关插件)，通过这个通道我们才能在通过远程服务器来解释运行当前debug的程序。

准备

  • 一台安装了xdebug插件的服务器(Homestead默认安装了)
  • PHPStorm

IDE设置

切回到IDE依次Run->Web Server Debug Validation 对话框如下

可以看到验证时有两种服务器选择，其实两者都能验证，但是对于laravel这种对于url重写的web程序来说我们最好选择前者，不然验证脚本的访问会有一些问题。

选择local的话，如果你使用的是laravel框架一定要记住路径一定是project根目录加上public目录，不然浏览器无法访问到非public目录文件(其它框架同理)，然后是填写到验证脚本的url，大家可以看到我填写了一个xdebugdemo.app 这是我通过yaml文件配置到Homestead里面去的一个本地域名，一个虚拟机里面多个项目同时部署时只能通过端口或者域名的方式把请求分发到不同的项目，因为我homestead里面部署了不少项目所以我填写了这个项目的url，对你们而言不管是什么项目什么框架只要你填写的url能访问到PHPStorm部署上去的脚本那就可以点击验证。

点击validate，我们可以看到对话框拿到了很多信息，xdebug版本号以及刚才我们在服务器编辑的xdebug配置文件里面的内容，而且没有报错，如果你看到了这些证明你的配置离成功已经不远了。

为了给大家演示，我这个debugdemo也是直接laravel new出来的一个新项目，我们从头演示如何设置断点开启debug模式。

大家可以看到我的项目还没有配置任何的运行选项，注意我红圈圈出来的地方，我们点击三角形下的编辑选项通过+号选择一个php http request

我们可以看到就是定义了一个HTTP请求的一些基本属性，对我演示而言我只需要配置一个服务器，其它的东西我们暂时不用管，这样就完成了一个对服务器web根目录GET请求的配置

服务器的配置一定要注意一个目录映射的问题，也就是本地文件的路径对应服务器上文件的路径应该有一个映射关系，不然debug程序无法跟到对应的代码里去。

保存修改后，回到PHPStorm主界面我们可以看到又上角多了一个绿色的三角形按钮以及debug按钮

如果点击运行的话，PHPStorm就会根据我们的HTTP请求配置进行一次HTTP请求并把服务器返回的结果打印在控制台上，我们现在主要演示debug，先打断点，点击行号右侧区域红点就会停留在代码左侧，然后点击debug按钮，可以看到我们的运行已经停在了36行说明断点运行正常。

可能遇到的问题

  • 断点没有停下 首先确保你的web server debug validation通过了验证，这个很关键。
  • 如果确保debug validation没有任何问题，很有可能是目录映射的问题。首先切到phpstorm->run->break at first line in php scripts,这样无论你断点打在何处debug程序都会在第一行代码停下来。如果这样你的程序是能停下来的说明十有八九是你目录映射出现了问题。
  • validation的时候卡住了，这个坑我也踩过，当然我不保证我们是一个问题。之前说过phpstorm可以设置一个IDE全局的代理，如果你设置过那很有可能就是这个问题了。因为你的自定义域名走了代理通道导致read timeout。去proxy代理设置把no proxy for那一栏加上你的自定义域名。当然你通过代理软件pass也是可以的。

总结

其实整个配置过程没有难点有的只是繁琐，大多数人又喜欢捷径，对于断点更是不屑感觉还是echo大法好，var_dump大法好，殊不知软件开发的终极奥义就是创造工具和使用工具。
